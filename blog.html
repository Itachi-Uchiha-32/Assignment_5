Question-2: What is the difference between innerHTML, innerText, and textContent ?
innerHTML: Gets or sets the HTML content inside an element (including HTML tags). Allows HTML markup (tags like , , etc.) innerText: Gets or sets the visible text content of an element, excluding HTML tags textContent: Gets or sets the text content of an element, including hidden text (no HTML parsing) Unlike innerText, it does not consider CSS styling
Question-3: What is event delegation in the DOM?
Event delegation is a technique where instead of adding event listeners to each individual element, a single event listener is added to a parent element, This listener listens for events that bubble up from child elements, It uses the event's target property to determine which child element triggered the event, It improves performance, especially when dealing with a large number of elements, It allows dynamically added elements to inherit the event listener without having to rebind the listener,
Question-4: What is event bubbling in the DOM?
Event bubbling is the process by which an event, after being triggered on an element, propagates (or "bubbles") up to its parent elements in the DOM tree, This means the event is first captured by the target element and then passed up to its ancestors, triggering the same event on each parent element,
